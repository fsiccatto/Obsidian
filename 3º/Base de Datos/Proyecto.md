## 🗺️ Mapeo del Modelo y Relaciones

Tu base de datos está estructurada con una entidad central fuerte (**`MASCOTA`**), la cual tiene una clave primaria compuesta (`DNI_Dueno`, `NombreMascota`). Esto significa que las tablas hijas arrastran ambas columnas para poder relacionarse.

### 🏠 Entidades Base (Catálogos)

Estas tablas no dependen de nadie; existen por sí solas.

* **`DUENO`**: Guarda los datos de los clientes. Su clave es el `DNI`.
* **`MEDICO`**: Registro de veterinarios con su `Matricula`.
* **`TIPO_SERVICIO`**: Catálogo de prestaciones (Consultas, Guardias, etc.) con sus precios actuales.
* **`VACUNA`**: Medicamentos disponibles con su precio de venta actual.
* **`HABITACION`**: Espacios físicos para internación o guardería (`NroHabitacion`).

### 🐕 Entidad Principal Dependiente

* **`MASCOTA`**: Depende directamente de **`DUENO`**. Su clave primaria es de tipo identificativa (`DNI_Dueno` + `NombreMascota`). Si borras un dueño, sus mascotas desaparecen (si tuviera borrado en cascada).

### 📑 Entidades de Eventos / Historial (Tablas Hijas)

Todas estas tablas registran el "día a día" de la veterinaria y apuntan obligatoriamente a una mascota específica:

* **`CONSULTA`**: Relaciona a una `MASCOTA` con un `MEDICO` y un `TIPO_SERVICIO`. Su clave primaria incluye la `FechaHoraConsulta` porque una mascota puede ir a consulta muchas veces.
* **`NOTA_EVOLUCION`**: Es una hija directa de `CONSULTA`. Registra el seguimiento médico de una consulta específica.
* **`APLICACION_VACUNA`**: Une a la `MASCOTA` con la `VACUNA` aplicada en un momento exacto. Guarda el `PrecioVentaHistorico` (¡excelente decisión de diseño para que los aumentos de precios no te alteren el pasado!).
* **`HOSPEDAJE`**: Registra la estadía de una `MASCOTA` en una `HABITACION`. Si `FechaHoraSalida` es `NULL`, la mascota sigue adentro.

## 🔍 Queries de Práctica (De menos a más)

Para probar estas consultas, métete al **SQL Commands** de tu APEX, pégalas de a una y dale a **Run**.

### 1. Nivel Cachorro: Filtros y Ordenamiento básico

*¿Cuáles son las mascotas de la especie 'Perro' y qué edad aproximada tienen hoy?*

```sql
SELECT NombreMascota, Raza, FechaNacimiento,
       TRUNC(MONTHS_BETWEEN(SYSDATE, FechaNacimiento) / 12) AS Edad_Anos
FROM MASCOTA
WHERE Especie = 'Perro'
ORDER BY FechaNacimiento DESC;

```

### 2. Nivel Adulto: Joins Esenciales

*Traer el historial de consultas médicas: Nombre de la mascota, nombre del dueño, fecha de la consulta, qué médico la atendió y el motivo.*

```sql
SELECT d.Nombre AS Nombre_Dueno,
       c.NombreMascota,
       c.FechaHoraConsulta,
       m.Nombre AS Nombre_Medico,
       c.Motivo
FROM CONSULTA c
JOIN DUENO d ON c.DNI_Dueno = d.DNI
JOIN MEDICO m ON c.Matricula_Medico = m.Matricula
ORDER BY c.FechaHoraConsulta DESC;

```

### 3. Nivel Ovejero Alemán: Funciones de Agregación (`GROUP BY`)

*¿Cuánto dinero ha generado en total cada mascota sumando sus consultas ejecutadas?*

```sql
SELECT DNI_Dueno, NombreMascota, SUM(PrecioConsulta) AS Total_Gastado
FROM CONSULTA
GROUP BY DNI_Dueno, NombreMascota
ORDER BY Total_Gastado DESC;

```

### 4. Nivel Alpha / Certificado Oracle: Manejo de `NULL` y condicionales

*Ver el estado actual del Hospedaje: Mostrar qué mascotas están actualmente internadas/hospedadas (salida es NULL) y calcular cuántos días llevan allí.*

```sql
SELECT NombreMascota, NroHabitacion, FechaHoraEntrada,
       TRUNC(SYSDATE - FechaHoraEntrada) AS Dias_Internado,
       NVL(TO_CHAR(FechaHoraSalida), 'Sigue Hospedado') AS Estado_Salida
FROM HOSPEDAJE
WHERE FechaHoraSalida IS NULL;

```

## 💡 "Huevadas" y mejoras que se le podrían meter (Para cuando te aburras)

Revisando el código con ojo crítico de arquitectura de software, si este sistema escala, acá hay un par de detalles divertidos para sumarle:

1. **Las contraseñas y teléfonos:** Tienes la columna `TelefonoEncriptado` en `DUENO`. Si vas a encriptar datos en Oracle, te va a tocar jugar con el paquete nativo `DBMS_CRYPTO`. ¡Un lindo dolor de cabeza para investigar!
2. **El peligro de los nombres como clave primaria:** Tu tabla `MASCOTA` usa `NombreMascota` como parte de la `PRIMARY KEY`. Si un dueño tiene un perro llamado "Firulais", y el perro fallece o el dueño adopta otro y le pone "Firulais" también... vas a tener una colisión de datos histórica. Un campo `IdMascota NUMBER GENERATED BY DEFAULT AS IDENTITY` te salvaría la vida a futuro.
3. **Hacer un Trigger de Auditoría:** Podrías crear una tabla llamada `LOG_DEUDAS` y un Trigger en Oracle que salte cada vez que le modifiques el campo `DeudaActiva` a un `DUENO`, guardando cuánto debía antes y cuánto debe ahora.
